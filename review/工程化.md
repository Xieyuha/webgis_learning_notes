# 工程化
商业模式 -> 产品 -> `实现` -> 交付 -> 运营 -> 迭代
## 模块化     
- 函数
- 文件
- 模块：module，具有独立功能的代码单元  
## 包管理
- 包：package，一系列模块的集合
    - 开发依赖
    - 生产依赖
- 框架：framework，约束代码结构
- 库：library，提供特定功能的代码集合    
## 语言问题
### js 语言问题
- 兼容性
    - api兼容
        - polyfill(垫片/填充物)，模拟新api：core-js
    - 语法兼容/增强
        - syntax transpile(转译)，将新语法转换为旧语法：Babel,runtime
- 语言增强
    - 类型检查：TypeScript,Flow
    - 静态检查：ESLint,Prettier
### CSS 语言问题
- 语法缺失（变量、嵌套、函数等） 
    - 预处理器：Sass,Less,Stylus
    - 后处理器：PostCSS

- 功能缺失（颜色函数、数学函数等）
    - PostCSS 插件
