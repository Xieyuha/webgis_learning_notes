# 核心思路
*为什么*
0. 业务/场景背景
1. 我面临什么问题——表达什么信息
2. 难点是什么，实现的困难在哪里——数据（结构、精度、离散型）、表达、绘制
3. 有什么应对手段
4. 哪种手段可以使用
    - 实现复杂度
    - 性能开销
    - 可维护性、耦合性
5. 编写时怎样拆分
    - 数据处理
    - 中间结果
    - 表达层
    - 参数控制
6. 扩展
